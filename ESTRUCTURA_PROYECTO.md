# 📂 Estructura del Proyecto - Workshop SQL

Para que la profesora vea un repositorio profesional, mantendremos esta organización:

## 1. 📂 `sql/`
Aquí van todos los archivos con extensión `.sql`. 
- `punto1_estudiantes.sql`: Consultas de notas y rendimiento.
- `punto2_facultades.sql`: Consultas de estructura administrativa (Tu parte).
- `punto3_consolidado.sql`: La "Gran Tabla" y limpieza de datos.

## 2. 📂 `docs/`
Los documentos de análisis en formato Markdown (.md) o PDF.
- `analisis_B_facultades.md`: Tu propuesta administrativa.
- `analisis_A_estudiantes.md`: Propuesta de mejora académica.
- `informe_consolidado.md`: Conclusiones finales del Integrante C.

## 3. 📂 `assets/`
- `graficas/`: Imágenes de los diagramas de barras, tortas, etc.
- `capturas/`: Evidencia de las consultas corriendo en Oracle Live SQL.

## 4. 📂 `templates/`
Formatos base para que todos escriban igual.

---

## 📤 ¿Cómo enviar los archivos a tus compañeros?

**Opción A: Por Git (Recomendado para la nota)**
1.  Tú trabajas en tu rama `feature/facultades`.
2.  Subes tus archivos a esa rama: `git push origin feature/facultades`.
3.  Tu compañero A descarga tus cambios: `git fetch` y luego `git checkout feature/facultades`.
4.  Él podrá ver tu carpeta `sql/` y `docs/` con lo que tú avanzaste.

**Opción B: Pull Request (Lo que la profesora ama ver)**
1.  Cuando termines tu punto, crea un **Pull Request** en GitHub de tu rama hacia `main`.
2.  Tus compañeros revisan el código, comentan y le dan "Merge". Esto deja rastro de que hubo **trabajo en equipo y revisión de pares**.
